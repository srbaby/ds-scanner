- 尾盘价格更接近收盘价，减少滑点

### 🥇 三道金牌（前置硬门槛）

#### 第一道金牌：量能关（拒绝假突破）
```
标准：量比 ≥ 1.20（全天真实放量，含临界值1.20）
确认时间：以14:55的实时量比为准
早盘量比脉冲不算，尾盘确认才算
目的：过滤无量的假突破，确认资金真实进场
```

**📐 量比定义（数据字典，v2.5已内化）**
```
量比 = 当日全天成交量 / 过去5个交易日日均成交量

说明：
├─ 单位：成交量（手），不是成交额（元）
├─ 分母：取前5个完整交易日，不含当日
└─ 分子：全天完整成交量（非盘中累计）

量比含义：
├─ 量比 = 1.0：今日与近5日均值持平
├─ 量比 ≥ 1.20：有效放量 → 第一道金牌通过
└─ 量比 < 1.0：缩量，资金退潮
```

**⏱️ 盘中时间修正（扫描器已自动处理）**
```
问题：扫描器盘中运行时，拿的是"当日累计量"，比全天少
修正：实时量 × 时间折算系数 = 全天预估量

折算系数 = 240分钟（全天）/ 已交易分钟数

交易时段：9:30-11:30（120分钟）+ 13:00-15:00（120分钟）= 240分钟

关键时刻折算系数：
├─ 11:30（上午收盘）：×2.0
├─ 12:xx（午休）：  按上午满算，系数×2.0
├─ 14:30：          ×1.14
├─ 14:49（自动扫描时）：×1.05
├─ 14:55（确认时）：×1.02 ← 接近全天真实值
└─ 15:00（收盘）：  ×1.0，无需折算

⚠️ 对AI的说明：
扫描报告中的量比已完成时间折算。
AI直接用报告中的量比数值与1.2比对即可，无需重复计算。
```

**🚫 常见误区**
```
❌ 早盘量比高就是放量  → 折算系数大，9:30-10:00脉冲失真，不可信
❌ 量比对比昨日单日    → 应对比5日均量，不是昨日
❌ 量比用成交额计算    → 应用成交量（手），不是金额
❌ 1.5才算放量        → 本系统标准是1.2（v2.4已从1.5调低）
```

**📌 临界值说明**
```
量比 ≥ 1.20 即视为通过（含临界值）
扫描器输出精度为小数点后两位，1.20算通过，1.19不通过
```

### 🛡️ v2.6 三道防线止损体系

#### 第一道防线：逻辑止损（软止损）

```
触发条件（客观量化，非主观判断）：
├─ 扫描器policy总分（附录A，满分30）跌至 15分以下
└─ 说明板块政策逻辑已被证伪或出现重大利空

执行：当日尾盘14:55-15:00清仓，不等待价格止损

重要原则：
✅ 缩量阴跌 + policy总分≥15 → 坚定持有，无视日常波动
❌ policy总分跌破15 → 无论浮盈浮亏，立即执行逻辑止损
⚠️ 上班族无时间主观判断政策，扫描器policy总分是唯一客观触发器
   （勿与模块7四维评分的"政策催化30分"混用，后者仅用于开仓）
```

#### 第二道防线：绝对价格止损（硬止损）

```
触发条件：现价跌破买入成本价 × (1 - 8%)

执行（统一口径，2026-06-12澄清）：
├─ 盘中发现跌破（价格警报触发）→ 立即市价清仓，不等14:55（流程见模块5情况4）
├─ 盘中未能执行（开会等无法操作）→ 收盘确认跌破后，次日尾盘清仓兜底
└─ 一旦触发，必须无条件机械斩仓，不抱反弹幻想

说明：
├─ 面临系统性风险或基本面逻辑彻底证伪的兜底线
└─ 扫描器输出"距-8%止损线剩余空间：XX%"，方便每日确认

计算示例：
买入价 1.00 → 硬止损线 0.92
买入价 1.50 → 硬止损线 1.38
```

#### 第三道防线：时间止损（最终兜底）

```
触发条件：持仓满21个自然日（约一个月），仍未触发以上任何止损
执行动作：强制换股，释放资金，不论浮盈浮亏

逻辑：
├─ 主力资金一个月内洗不明白、拉不起来，说明趋势判断彻底失效
├─ 强制换股保证资金流动性，防止"短线波段"退化为"长期套牢"
└─ 保留资金利用率，等待下一个高质量信号

例外：浮盈>8% + 当前扫描器policy总分≥20 + 有明确新催化 → 可延长至28天，仅此一次
```

**优先级（由高到低）：**
```
价格止损（-8%，盘中可触发）
    ↓
逻辑止损（policy总分<15，尾盘触发）
    ↓
时间止损（21天，强制换股）
```

## ⚔️ 模块4：持仓-新信号冲突处理矩阵

### 冲突场景判定表

| 当前持仓状态 | 新信号类型 | 处理原则 | 量化标准 |
|------------|-----------|---------|---------|
| 浮亏 < -8% | 任何新信号 | 🔴 先止损，禁止开新仓 | 持仓触发价格止损→立即清仓 |
| 浮亏 -8% ~ -5% | 不同板块 & 评分≥80 | 🟡 可开10%新仓 | 但持仓必须设-10%警戒止损 |
| 浮亏 -8% ~ -5% | 其他信号 | ❌ 观察持仓，不开新仓 | 等待持仓止损或企稳 |
| 浮亏 -5% ~ 0% | 不同板块新信号 | 🟡 谨慎开仓 | 控制总仓位<60% |
| 同标的新信号 | 评分 > 首次买入+10分 | ✅ 可加仓10% | 信号显著增强，补仓合理 |
| 同标的新信号 | 评分提升 < 10分 | ⚠️ 不加仓 | 避免补仓陷阱 |
| 同板块新标的 | 板块持仓<30% & 评分≥75 | ✅ 可开5-10%小仓 | 板块分散，控制集中度 |
| 同板块新标的 | 板块持仓≥30% | ⚠️ 谨慎，一般不开仓 | 避免板块过度集中 |
| 浮盈 > 5% | 不同板块新信号 | ✅ 正常开仓 | 有利润垫，可承担新风险 |
| 浮盈 0-5% | 不同板块新信号 | 🟡 谨慎开仓 | 控制总仓位<60% |

---

### 冲突处理决策树

```python
def 持仓冲突判定(当前持仓, 新信号):

    # 第一层：价格止损优先
    if 当前持仓.浮亏 < -8%:
        return "🔴 立即止损，禁止开任何新仓"

    # 第二层：深度浮亏区谨慎
    if -8% <= 当前持仓.浮亏 < -5%:
        if 新信号.板块 != 当前持仓.板块 and 新信号.评分 >= 80:
            return "🟡 可开10%新仓，但持仓设-10%止损警戒"
        else:
            return "❌ 观察持仓，不开新仓"

    # 第三层：同标的判定
    if 新信号.标的 == 当前持仓.标的:
        if 新信号.评分 > 当前持仓.买入评分 + 10:
            return "✅ 信号显著增强，可加10%仓"
        else:
            return "⚠️ 信号提升不足，不加仓，避免补仓陷阱"

    # 第四层：同板块判定
    if 新信号.板块 == 当前持仓.板块:
        板块总仓位 = sum([p.仓位 for p in 所有持仓 if p.板块 == 新信号.板块])
        if 板块总仓位 < 30% and 新信号.评分 >= 75:
            return "✅ 板块分散，可开5-10%小仓"
        else:
            return "⚠️ 板块持仓过重，谨慎"

    # 第五层：不同板块判定
    if 当前持仓.浮盈 > 5%:
        return "✅ 有利润垫，可正常开仓"
    else:
        return "🟡 谨慎开仓，控制总仓位<60%"
```
