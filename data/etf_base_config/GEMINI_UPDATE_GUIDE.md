# ETF板块基础分配置文件 - Gemini更新指南

> 本文件说明评分规则和操作方法，**内容长期有效，无需随时事更新**。
> 时效性内容（当前分数、背景事件、修正原因）全部在 `etf_base_config.json` 中维护。

---

## 📋 文件说明

`etf_base_config.json` 记录各ETF板块的**政策基础分（0-15分）**。

脚本在基础分之上叠加技术分（0-8）和相对强度分（0-7），得出 `etf_pool.json` 中存储的 **policy总分（最高30分）**。

### ⚠️ v2.6止损联动
持仓ETF的 policy总分 < 15 时，脚本自动触发逻辑止损建议。
评分要基于事实——但低分判断有实际交易后果，需认真对待。

---

## 🎯 基础分含义（0-15分）

| 分段 | 含义 | 典型板块 |
|------|------|---------|
| 12-15 | 国家战略级 | 半导体、AI算力、创新药 |
| 9-12 | 重点支持级 | 新能源、科创、战略物资 |
| 6-9 | 稳增长级 | 消费、医药、普通周期 |
| 4-6 | 中性周期级 | 光伏、金融科技、港股科技 |
| 2-4 | 政策工具级 | 银行、证券 |
| 0-2 | 政策限制级 | 房地产、教培（当前池中无） |

---

## 📊 评分参考维度

| 维度 | 满分 | 说明 |
|------|------|------|
| 国家战略地位 | 5 | 是否写入五年规划、政府工作报告；是否涉及自主可控/国家安全 |
| 产业政策力度 | 5 | 财政补贴、税收优惠、产业基金、产能引导 |
| 供需格局导向 | 3 | 去产能vs扩产能；需求刺激vs抑制 |
| 监管态度 | 2 | 鼓励创新vs严格监管 |

---

## 💡 评分原则

1. **基于政策事实，不基于市场情绪或近期涨跌**
2. **重大变化才调整（±2分以上），无明确政策依据不动**
3. **只修改 `scores` 内的数值，不改变JSON结构**
4. **每次修改必须在 `_update_log` 中留记录，并更新 `_meta.version`**

---

## 🚀 操作流程

```bash
# 以下命令均在 X-Plan/ 根目录下执行

# 步骤1：将 data/etf_base_config.json + PROMPT_FOR_GEMINI.md 发给 Gemini
# Gemini 返回更新后的 JSON

# 步骤2：覆盖配置文件
cp 新文件.json data/etf_base_config.json

# 步骤3：验证 JSON 格式
cat data/etf_base_config.json | python3 -m json.tool

# 步骤4：强制刷新 policy 分数
python3 automation/ds_scanner.py --refresh-policy

# 步骤5：查看报告，确认分数已更新
```
