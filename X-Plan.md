# X-Plan v3.1 — 右轮 ETF 波段目标仓位系统

> 版本日期：2026-07-09  
> 性质：独立影子交易验证系统方法论，不与主投资体系混同。  
> 日常执行 Prompt：`Prompt.md`。若二者冲突，以本文为准。
> 统一版本清单：`VERSION.json`。

---

## 0. 一句话定义

X-Plan 是一个“精选 ETF 右轮波段目标仓位系统”。它每天只做一次尾盘扫描和一次最终操作清单，不做盘中多次微调。系统目标不是高频交易，而是在较高胜率基础上抓住 ETF 波段爆发力：看准时给足确定仓位，走弱时及时降仓或换到更强标的。

核心决策不再是“买不买”，而是：

```text
每只 ETF 今天应该是 0%、10%、15%、20% 哪一档目标仓位？
资金今天应该继续留在原标的，还是从弱标的流向更强标的？
```

---

## 1. 系统事实与执行边界

1. ETF 池是精选代表池，不预设同类标的互相确认。
2. 每天只有一次扫描、一次扫描器权威决策、一次人工执行窗口。
3. 扫描器操作清单是当日终局结果；AI 只做独立审计，不得覆盖决策。
4. 单只 ETF 自动目标仓位最高 20%。
5. 总权益仓位上限 65%。
6. 仓位百分比统一按账户总市值计算。
7. 加仓只允许右轮强势确认，不允许浮亏补仓。
8. 卖出不只来自止损，也来自趋势走坏、盈利钝化、时间失败和横向择优换仓。

### 仓位口径

```text
账户总市值 = 可用现金 + 全部 ETF 持仓市值
单只 ETF 仓位% = 单只 ETF 当前市值 / 账户总市值
总权益仓位% = 全部 ETF 持仓市值 / 账户总市值
```

禁止使用以下口径替代：

```text
旧 20 万虚拟本金
买入成本
仅持仓市值作为分母
主体系资产口径
```

## 2. 每日目标仓位状态机

系统只允许四个目标仓位档：

```text
0% / 10% / 15% / 20%
```

不允许使用“酌情”“小仓”“重仓”“10%-15%”“20%-30%”等浮动表述。

| 今日状态 | 固定条件 | 今日目标仓位 | 动作 |
|---|---|---:|---|
| 无有效信号 | 未通过 B/A/S 任一等级，且无持仓保留理由 | 0% | SKIP / SELL |
| B 普通信号 | 达到 B_SIGNAL | 10% | BUY / HOLD / REDUCE |
| A 强信号 | 达到 A_SIGNAL | 15% | BUY / ADD / HOLD |
| S 爆发信号 | 达到 S_SIGNAL | 20% | BUY / ADD / HOLD |
| 过热强势 | 强但 MA20 偏离过大 | 原仓位档，不上调 | HOLD |
| 盈利钝化 | 已盈利但量能、资金或评分降级 | 降一档 | REDUCE |
| 趋势破坏 | 右轮条件不再成立 | 0% | SELL |
| 更优替代 | 原持仓降级，新标的明显更强 | 原标的降仓或清仓，新标的建仓 | REDUCE/SELL + BUY |
| 风控失败 | 价格止损、综合逻辑分失效、数据异常 | 0% | SELL |

### 动作由目标仓位差决定

| 当前实际仓位 | 今日目标仓位 | 动作 |
|---:|---:|---|
| 0% | 10% | BUY 到 10% |
| 0% | 15% | BUY 到 15% |
| 0% | 20% | BUY 到 20% |
| 10% | 15% | ADD 到 15% |
| 15% | 20% | ADD 到 20% |
| 20% | 15% | REDUCE 到 15% |
| 15% | 10% | REDUCE 到 10% |
| 任意 | 0% | SELL 全部 |

执行容差：

```text
如果实际仓位与今日目标仓位差值 < 2 个百分点，可输出 HOLD，
但仍需写明今日目标仓位和当前实际仓位。
```

## 3. B/A/S 信号分级

所有升级条件必须来自 `report.txt` 已给字段；缺关键字段时不得脑补，只能降级。

### 公共硬条件

所有 B/A/S 新开仓或加仓必须同时满足：

```text
现价 > MA20
MA20 偏离度 <= 15%
总权益仓位预计不超过 65%
单只 ETF 今日目标仓位不超过 20%
无优先级更高的持仓 SELL/REDUCE 任务未处理
```

其中：

```text
MA20 偏离度 = 现价 / MA20 - 1
相对沪深300强度 = ETF 当日涨跌幅 - 沪深300当日涨跌幅
```

### B_SIGNAL：普通右轮信号 → 10%

必须同时满足：

```text
四维评分 75-79
量比 >= 1.20
现价 > MA20
MA20 偏离度 <= 15%
资金流向不是“流出”
```

### A_SIGNAL：强右轮信号 → 15%

必须同时满足：

```text
四维评分 80-84
量比 >= 1.26
现价 > MA20
MA20 偏离度 <= 15%
资金流入，或相对沪深300强度 >= +1.0%
```

### S_SIGNAL：爆发级右轮信号 → 20%

必须同时满足：

```text
四维评分 >= 85
量比 >= 1.26
现价 > MA20
MA20 偏离度 <= 12%
资金流入
相对沪深300强度 >= +1.5%
RSI <= 75
```

### 过热处理

```text
12% < MA20 偏离度 <= 15%：
  可保持 A/B 目标仓位，但不得升级到 S。

15% < MA20 偏离度 <= 20%：
  禁止新开仓和加仓；已持仓若趋势未坏，输出 OVERHEAT_HOLD。

MA20 偏离度 > 20%：
  禁止新开仓和加仓；若已有浮盈且资金/评分转弱，按 PROFIT_WEAKEN 降仓。
```

## 4. 开仓、加仓、减仓、清仓

### 开仓

开仓必须给出确定目标仓位：

```text
B_INITIAL_BUY：目标 10%
A_INITIAL_BUY：目标 15%
S_INITIAL_BUY：目标 20%
```

若多个新标的同时达标，先做横向排序，再在总仓位 65% 上限内选择最强者。不得为了提高资金利用率而买入低等级信号。

### 加仓

加仓不是补亏，是右轮确认。

允许 ADD 的条件：

```text
当前持仓浮盈 >= 0
今日信号等级高于当前实际仓位对应档
现价 > MA20
MA20 偏离度 <= 15%
资金流向未转弱
预计单只仓位 <= 20%
预计总权益仓位 <= 65%
```

规则代码：

```text
A_CONFIRM_ADD：当前约 10%，今日 A，目标 15%
S_CONFIRM_ADD：当前约 10%或15%，今日 S，目标 20%
```

禁止：

```text
浮亏加仓
趋势走弱加仓
偏离过热加仓
仅因现金多而加仓
```

### 减仓

盈利本身不是卖出理由；趋势不再配得上原仓位，才减仓。

规则代码：

```text
SIGNAL_DOWNGRADE：今日有效信号等级低于当前目标档，直接降到今日等级对应的固定目标仓位。
PROFIT_WEAKEN：已有浮盈，但量能、资金、评分或趋势降级，目标仓位降一档。
OVERHEAT_HOLD：强势但偏离过大，只持有不加仓，不主动减仓。
SWITCH_OUT：原持仓降级且新标的明显更强，释放资金给更强标的。
```

优先口径：

```text
能够明确重新评级为较低 B/A 档 → SIGNAL_DOWNGRADE，目标直接对应 10%/15%。
尚不能形成较低有效等级，但已有浮盈且动能明确转弱 → PROFIT_WEAKEN，只降一档。
```

减仓只能降到下一档：

```text
20% → 15%
15% → 10%
10% → 0%
```

### 清仓

规则代码：

```text
RISK_STOP：触发 -8% 硬止损、综合逻辑分 < 15、数据异常或重大风险。
TREND_BREAK：跌破 MA20 且资金转弱，或评分跌破 60。
TIME_FAIL：持仓到期但未保持 A/S 强势。
SWITCH_OUT：原标的已不强，且新标的明显更优，需要清出资金。
```

## 5. 横向择优与换仓

每天必须同时比较：

```text
现有持仓
观察池全部候选
报告中出现的非池内持仓或候选
```

排序优先级固定为：

```text
信号等级：S > A > B > 无效
四维评分
相对沪深300强度
量比
资金流向
MA20 偏离度是否过热
```

### SWITCH_OUT 条件

允许卖弱换强的条件：

```text
新标的为 A 或 S；
原持仓今日等级低于新标的至少 1 档，或原持仓触发 PROFIT_WEAKEN / TREND_BREAK；
新标的评分比原持仓高 >= 8 分，或新标的是 S 且原持仓不是 S；
换仓后单只仓位 <= 20%，总权益仓位 <= 65%；
操作清单同时写出旧标的降级原因和新标的更优原因。
```

不得换仓的情况：

```text
新标的只是同级略强，评分差 < 8 分；
原持仓仍为 S 且未过热；
换仓会突破总仓位上限；
需要卖出风控标的但又买入低等级标的。
```

## 6. 仓位上限与资金利用

### 硬约束

```text
单只 ETF 自动目标仓位 <= 20%
总权益仓位 <= 65%
每日只输出一张最终操作清单
```

### 上涨漂移

```text
因持仓上涨导致总权益仓位自然漂移超过 65%：
  不因“超过65%”单独强制卖出。
  但禁止任何新增 BUY/ADD。
  只有当标的触发 PROFIT_WEAKEN、TREND_BREAK、SWITCH_OUT 或 RISK_STOP 时才降仓。
```

### 资金利用反馈

若系统胜率持续高于 70%，但总权益仓位长期低于 30%，属于方法论反馈信号，优先检查：

```text
强信号开仓是否过小
确认加仓是否过严
盈利钝化是否过早
换仓规则是否过于保守
```

## 7. 评分体系

四维评分总分 100，由扫描器确定性计算，只用于开仓、加仓、持仓等级和横向排序。AI 只做审计，不得修改评分或操作。

### 政策催化：30分

```text
由 `data/etf_base_config.json` 的基础分（0-15）乘以 2，并限制在 0-30。
```

### 技术面：25分

```text
MA20位置 10分 + 量比 6分 + 资金流 5分 + RSI 4分。
```

### 情绪与强度：20分

```text
相对沪深300强度 10分 + 量价动能 6分 + RSI拥挤度 4分。
```

### 风险收益比：25分

```text
使用 ATR14：止损取 MA20-0.5ATR、现价-2ATR、现价-8% 中的最高有效价，
有效风险至少 1ATR；目标价为 max(20日高点, 现价)+2ATR。
赔率 >3、2-3、1.5-2、<1.5 分别计 25、20、14、8 分。
```

## 8. 持仓生命周期与时间规则

时间规则不再机械覆盖强势标的。

```text
持仓满约 21 个自然日（约 15 个交易日）：
  若今日仍为 A 或 S，允许继续持有，但必须记录 TIME_EXTEND。
  若今日为 B 或无效，触发 TIME_FAIL，目标仓位降到 0%。
```

价格风控：

```text
浮亏 <= -8%：RISK_STOP，SELL 全部。
浮亏 < 0：禁止 ADD。
浮亏但未到 -8%，若趋势仍有效，可 HOLD；若趋势破坏，TREND_BREAK。
```

盈利处理：

```text
浮盈扩大且今日仍为 A/S：HOLD，不因盈利本身减仓。
浮盈 >= 5% 且今日降为 B 或资金转弱：PROFIT_WEAKEN，降一档。
浮盈 >= 8% 且跌破 MA20 或评分 < 75：SELL 或 REDUCE 到 0%-10%。
```

## 10. 反馈治理

反馈不是周末复盘，而是基于 Gist 数据的规则验证。

后续统计必须能按 `rule_code` 聚合：

```text
S_INITIAL_BUY 胜率、平均收益、最大回撤
A_INITIAL_BUY 胜率、平均收益、最大回撤
A_CONFIRM_ADD / S_CONFIRM_ADD 加仓后收益贡献
PROFIT_WEAKEN 减仓后是否频繁卖早
SWITCH_OUT 换仓后是否跑赢原持仓
TIME_FAIL 是否卖早
RISK_STOP 是否来自执行延迟或信号失效
```

治理触发：

```text
胜率高、资金利用率低：检查开仓/加仓是否过保守。
ADD 后频繁回撤：强势确认条件过松。
REDUCE 后频繁大涨：盈利钝化规则过早。
SWITCH_OUT 后跑输原持仓：横向择优门槛过低。
单笔大亏扩大：优先检查执行延迟、数据异常和退出链路，不直接归因于开仓仓位。
胜率明显跌破 60%-65%：收缩 S/A 档触发条件，暂停放大仓位。
```

## 11. 扫描器 policy 总分实现

扫描器 `policy` 总分满分 30，是持仓逻辑风控与横向排序辅助字段，不能与四维评分中的“政策催化30分”混用。

```text
base分：来自 data/etf_base_config.json，按主题政策倾向手动维护
tech分：价格/MA20位置 + RSI + 量比
strength分：ETF 涨跌幅相对沪深300的超额强度
逻辑风控阈值：policy 总分 < 15 → RISK_STOP
```
