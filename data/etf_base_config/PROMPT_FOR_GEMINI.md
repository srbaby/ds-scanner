# 给Gemini的提示词（直接复制使用）

---

## 📋 任务提示词

```
【任务】复盘并更新ETF板块政策基础分配置文件

我附上了 etf_base_config.json，请基于当前最新政策动态，判断是否需要调整各板块基础分。

文件内的 _update_log 和 _修正说明 记录了历史背景，请以此为基准判断变化。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【评分规则】

基础分含义（0-15分）：
• 12-15分：国家战略级（芯片/AI/创新药/战时核心军工）
• 9-12分：重点支持级（新能源/科创/战略物资）
• 6-9分：稳增长级（消费/医药/普通周期）
• 4-6分：中性周期级（光伏/金融科技/港股科技）
• 2-4分：政策工具级（银行/证券）

评分依据：
• 基于政策事实，不基于市场情绪或近期涨跌
• 有明确政策依据才调整（±2分以上），否则维持不变
• 只修改 scores 内的数值，不改变 JSON 结构

⚠️ 注意：本系统v2.6新增逻辑止损联动——持仓ETF的policy总分（基础分+技术分+强度分）< 15
时会触发止损建议。低分判断有实际交易后果，请认真对待。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【信息来源】

优先参考：
1. 最新重大政治/地缘事件（停战？升级？新冲突？）
2. 政府工作报告 / 发改委 / 工信部最新政策文件
3. 近1个月重大产业政策新闻

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【输出要求】

1. 返回完整 JSON（保持原结构，只修改 scores 数值）
2. 更新 _meta.version（如 2.5-WAR → 2.5-WAR-R1）
3. 在 _update_log 添加记录（日期 + 具体变更）
4. 文字说明：
   • 调整项：从X分→Y分，依据是什么
   • 维持不变的板块：一句话原因
```

## 💬 简化版（极简）

```
复盘ETF板块政策基础分（0-15分）。

附：etf_base_config.json（含历史背景和当前分数）

规则：
• 有明确政策依据才调整（±2分以上）
• 只改 scores 数值，保持 JSON 结构
• 更新 _update_log 和 _meta.version
• ⚠️ policy总分<15触发持仓止损，低分有实际后果

输出：完整 JSON + 变更说明
```

## 📤 使用步骤

```bash
# 以下命令均在 X-Plan/ 根目录下执行

# 1. 发给 Gemini：data/etf_base_config.json + 上面的提示词

# 2. 收到更新后的 JSON，覆盖文件：
cp 新文件.json data/etf_base_config.json

# 3. 验证格式：
cat data/etf_base_config.json | python3 -m json.tool

# 4. 刷新 policy 分数：
python3 automation/ds_scanner.py --refresh-policy
```
